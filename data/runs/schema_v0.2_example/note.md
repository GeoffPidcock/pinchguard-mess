# schema_v0.2_example

Committed reference run so collaborators on Concerns B/C/D can inspect
concrete rows and activations without rerunning the model. Generated by
`scripts/generate_example_run.py`, which drives the inference shim against
Qwen2.5-0.5B-Instruct on CPU.

## What's here

- `traces.jsonl` — one row per chat-completion call (schema v0.2; see
  `AGENTS.md` §3 for the canonical example).
- `activations/<step_id>.npz` — residual-stream hidden states keyed
  `L<layer_idx>`, shape `(1, 896)`, float16, captured at the `last_input`
  token position. `step_id` ↔ npz parity holds (AGENTS.md §4 invariant #1).

## Why "v0.2_example"

The data **will change** as the project progresses. Expect non-backwards-
compatible edits when any of the following land:

- **OpenClaw integration** (next phase) — once we see what fields the
  OpenClaw telemetry sidecar emits, several null-today fields will be
  populated (`turn_id`, `scenario_id`, `agent_id`, `agent_persona`,
  `context`, `observation`, `parsed_action`). The shape of `observation`
  and `parsed_action` is informed-guess only until then.
- **Concern C feedback** — layer selection (`LAYERS`), token position
  (`TOKEN_POSITION`), and any additional capture sites (attention output,
  MLP output, pre/post layernorm) get decided by the probe team after they
  see what's tractable. The current `[12, 23]` choice is a placeholder.
- **Concern B feedback** — fields the LLM-judge rubric needs but the
  current row doesn't carry will land as additions (e.g., scoped prompt
  segmentation, structured output parsing).
- **Concern D feedback** — feature surface the predictor needs (token
  counts, lexical flags) may either be precomputed and stored, or kept
  derived. Likely additions to the row, not edits to existing fields.

Bump the `schema_version` string and the directory name (`schema_v0.3_example`,
etc.) when a non-backwards-compatible change ships. Old example runs may be
retained as fossils for migration testing or removed once obsolete.

## Regenerating

```bash
uv sync --extra dev --extra capture
uv run python scripts/generate_example_run.py
```

This overwrites `traces.jsonl` and `activations/*.npz` in place. Inspect
the diff against git to see what changed.

## What this run is **not**

- Not a real scenario. The prompts are five hand-written executive-assistant
  queries, not an OpenClaw scenario YAML.
- Not statistically meaningful. Five rows is a smoke-sized sample for
  schema-shape sanity, not for probe training or rubric calibration.
- Not on production hardware. CPU Qwen2.5-0.5B activations are useful for
  plumbing tests but not for evaluating real drift signals — that work
  happens on Vast GPU in a later phase.
